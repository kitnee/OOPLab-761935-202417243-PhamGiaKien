Question 1: If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this? 
Answer: No, Java does not allow this. Both constructors would take a single String argument, resulting in the exact same signature: DigitalVideoDisc(String). In Java, overloaded methods or constructors must have different signatures (different number of parameters, or different types of parameters).

Question 2: Try to add a method addDigitalVideoDisc which allows to pass an arbitrary number of arguments for dvd. Compared to an array parameter. What do you prefer in this case? 
Answer: I prefer using varargs (e.g., addDigitalVideoDisc(DigitalVideoDisc... dvds)). It is more convenient and flexible because the caller can pass comma-separated objects directly without explicitly creating an array object (like new DigitalVideoDisc[]{dvd1, dvd2}).


Question 3: Is JAVA a Pass by Value or a Pass by Reference programming language? 
Answer: Java is strictly a Pass-by-Value programming language. When an object is passed to a method, a copy of the reference (the memory address) is passed, not the actual object itself nor the original reference variable.

Question 4: After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two objects still remain? After the call of changeTitle(...) why is the title of the JungleDVD changed? 
Answer: > - The swap method fails to change the original objects because it only swaps the local copies of the references (o1 and o2) inside the method. The original variables in the main method still point to their original objects.
The changeTitle method succeeds because it uses the copied reference to invoke a mutator method (setTitle). Since the copied reference points to the exact same object in the heap as the original reference, modifying the object's internal state (its title) reflects globally.
